# 冷知识语音播放器设计

> 日期：2026-04-03 | 交付：2026-04-03

## 概述

将现有单条冷知识翻转卡片升级为**多条冷知识 + 语音朗读 + 卡片同步播放器**。每个景点 5 条冷知识，点击 ? 展开播放器，支持自动朗读 + 手动翻页 + 进度条。

**用户：** 暖暖（8岁）为主要使用者
**技术栈：** 现有单文件 HTML + Web Speech API（纯前端）
**不需要 Firebase**

## 交互流程

1. 点击景点旁 **?** → 当前行下方展开冷知识播放器
2. 播放器显示：标题 + 进度条 + 当前冷知识 + 控制按钮
3. 点 **▶ 播放** → 语音朗读当前条 → 念完自动下一条 → 5 条全部念完自动停止
4. 点 **⏸** → 暂停语音和自动翻页
5. 点 ◀/▶ → 手动切换上/下一条
6. 再点 **?** 或点关闭 → 收起播放器，停止语音

### 样式

- 背景：`linear-gradient(135deg, #fff9e6, #fff0f5)`，金色 2px 边框，圆角 14px
- 进度条：粉色渐变填充 `#ffc8dd → #d63384`，高 6px，圆角，分 5 段
- 当前朗读段有呼吸闪烁动画
- 控制按钮：粉色圆形 36px，tap 友好
- 文字：14px，`#5a4830`，行高 1.6
- 卡片切换：fadeIn 动画 0.3s

## Web Speech API

```js
const utterance = new SpeechSynthesisUtterance(text);
utterance.lang = 'zh-CN';
utterance.rate = 0.9;  // 稍慢，适合小朋友听
utterance.onend = () => { /* 自动播下一条 */ };
speechSynthesis.speak(utterance);
```

- `speechSynthesis.pause()` / `resume()` / `cancel()` 控制暂停/恢复/停止
- iOS Safari 要求用户交互后才能首次播放（点按钮触发即可）
- 兼容性：iOS Safari 7+, Android Chrome 33+, 微信内置浏览器均支持

## 改动范围

### 删除

- 现有 `FUN_FACTS` 单条字符串对象
- 现有 flip card 创建逻辑（`flip-row` 相关）
- `.flip-back`, `.flip-inner`, `.flip-container` CSS（替换为新播放器样式）

### 新增

- `FUN_FACTS` 改为数组格式（每景点 5 条）
- 播放器 CSS（`.fact-player`, `.fact-progress`, `.fact-controls` 等）
- 播放器 JS（创建播放器 DOM、Web Speech API 控制、进度条更新、卡片切换动画）
- ? 气泡点击逻辑改为展开播放器

### 保留

- `.flip-trigger`（? 气泡）样式和注入逻辑（过滤打车/前往等行）
- 卡片展开的位置逻辑（在对应行下方插入）

## 验收标准

1. 每个景点 ? 气泡点击展开播放器，显示 5 条冷知识
2. 点播放 → 语音朗读 + 卡片同步切换 + 进度条推进
3. 念完 5 条自动停止
4. 暂停/恢复正常工作
5. 手动翻页正常
6. 手机端排版正常（max-width 480px）
7. 关闭播放器时语音停止
