# 行程页面互动功能设计

> 日期：2026-04-02 | 交付：2026-04-03（出发前一天）

## 概述

为北京旅行行程 HTML 页面增加 3 个互动功能，提升暖暖（8岁）的参与感和家庭温暖感。

**用户：** jumpin（手机）、张奕（手机）、暖暖（iPad）
**技术栈：** 现有单文件 HTML + Firebase Realtime DB + localStorage 兜底
**部署：** GitHub Pages（已有）

## 功能 1：景点集章打卡

### 交互

1. 页面底部固定导航栏，左侧"集章册"按钮（卡比抱星星）
2. 点开弹出集章册浮层，8 个圆形章位网格排列（2×4）
3. 未打卡：灰色虚线圈 + 景点名（灰字）
4. 点击章位 → 弹出确认气泡："暖暖到 XX 啦？" → 确认/取消
5. 确认 → 盖章动画（卡比星星旋转落下，0.8s）→ 章位变彩色
6. 长按已盖的章（1.5s）→ 弹出确认 → 可撤销（防误触需要，但保留仪式感）
7. 集齐 8 个 → 全屏彩蛋：卡比跳舞 + 星星雨 + "暖暖的北京大冒险 - 全部通关！"

### 数据模型

```json
// Firebase path: /trips/beijing2026/stamps
{
  "yiheyuan": { "done": true, "time": "2026-04-04T16:30:00" },
  "mutianyu": { "done": false },
  "niujie": { "done": false },
  "gugong": { "done": false },
  "guobo": { "done": false },
  "tiananmen": { "done": false },
  "beida": { "done": false },
  "yonghegong": { "done": false }

```

### 同步策略

- 写入时：先写 localStorage，再写 Firebase
- 读取时：Firebase 实时监听（`onValue`），更新本地 UI
- 离线时：localStorage 兜底，联网后 Firebase 自动同步
- 冲突解决：以最新时间戳为准

## 功能 2：妈妈/爸爸小纸条

### 交互 - 写（爸爸/妈妈端）

1. 底部导航栏右侧"写纸条"按钮（铅笔图标）
2. 点开 → 输入框（限 50 字）+ 署名选择（爸爸/妈妈 toggle）
3. 点"送出" → 信封飞走动画 → 写入 Firebase

### 交互 - 读（暖暖端）

1. 有未读纸条时 → header 下方出现粉色信封，轻轻晃动
2. 点击信封 → 信封打开动画（0.6s）→ 展开小纸条
3. 纸条样式：浅黄色纸张背景、手写体风格、边角微卷、署名 + 时间
4. 点"收好" → 纸条收入纸条盒
5. 纸条盒入口在导航栏或 header 区域，可回看所有历史纸条

### 数据模型

```json
// Firebase path: /trips/beijing2026/notes
{
  "note_1712345678": {
    "content": "暖暖今天爬长城好棒！",
    "from": "爸爸",
    "time": "2026-04-05T20:00:00",
    "read": false
  }
}
```

### 设计细节

- 每条纸条带时间戳，纸条盒按天分组
- 未读标记：信封 + 红点数字角标
- 旅行结束后纸条盒变成纪念册，可以翻看

## 功能 3：景点冷知识翻转

### 交互

1. 每日行程卡片中，每个景点行右侧加一个 "?" 小气泡（粉色圆形，卡比风格）
2. 点击 → 对应区域 CSS 3D flip 翻转（0.5s）→ 背面显示冷知识卡
3. 冷知识卡样式：星星边框 + 💡 图标 + 内容文字
4. 再点 → 翻回正面

### 技术

- 纯前端实现，内容硬编码在 HTML
- CSS `transform: rotateY(180deg)` + `backface-visibility: hidden`
- 不需要 Firebase

## 共用基础设施

### 底部固定导航栏

```
┌─────────────────────────────┐
│  ⭐ 集章册    ✏️ 写纸条    📮 纸条盒  │
└─────────────────────────────┘
```

- `position: fixed; bottom: 0`
- 卡比粉色半透明背景 + 毛玻璃效果
- 页面 body 底部留 padding 避免遮挡内容

### Firebase 配置

- 使用 Firebase Realtime Database（免费 Spark 计划）
- 数据路径：`/trips/beijing2026/`
- 无需认证（简单场景，URL 不公开，数据不敏感）
- SDK：用 Firebase JS SDK CDN 引入（compat 版本，体积小）

### 离线兜底

- 所有数据同时写入 localStorage
- Firebase `onValue` 监听实时更新
- 网络恢复后 Firebase 自动同步

## 文件结构

保持单文件 HTML 结构（当前架构），在 `<script>` 中增加：
- Firebase 初始化
- 集章打卡逻辑
- 小纸条读写逻辑
- 翻转卡交互逻辑

在 `<style>` 中增加：
- 底部导航栏样式
- 集章册浮层样式
- 信封/纸条动画
- 3D 翻转动画
- 彩蛋动画

## 验收标准

1. 三台设备打开同一 URL，打卡/写纸条实时同步
2. 集齐 8 章触发彩蛋动画
3. 长按可撤销已盖的章
4. 写纸条后暖暖端出现信封动画
5. 冷知识卡片翻转流畅
6. 离线时功能可用，联网后自动同步
7. 手机端排版正常（max-width 480px）
